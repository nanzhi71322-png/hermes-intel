# Environment Variable Documentation

The current project reads only two environment variables directly, both from `/opt/hermes/.env` through `python-dotenv`.

```python
load_dotenv("/opt/hermes/.env")
```

## Required Variables

### `TELEGRAM_BOT_TOKEN`

Required.

Telegram bot token generated by BotFather.

Used by:

```python
ApplicationBuilder().token(BOT_TOKEN).build()
```

Example:

```bash
TELEGRAM_BOT_TOKEN=1234567890:telegram-token-value
```

If missing or invalid, the bot cannot start polling Telegram successfully.

### `DEEPSEEK_API_KEY`

Required.

API key for DeepSeek's OpenAI-compatible chat completion endpoint.

Used by:

```python
OpenAI(
    api_key=DEEPSEEK_API_KEY,
    base_url="https://api.deepseek.com"
)
```

Example:

```bash
DEEPSEEK_API_KEY=sk-your-deepseek-key
```

If missing or invalid, chat, page analysis, intelligence scans, autonomous loops, and agents will fail when they call the LLM.

## Hard-Coded Runtime Configuration

These settings are not currently environment variables, but they behave like production configuration.

| Setting | Current value | Used for |
| --- | --- | --- |
| Env file | `/opt/hermes/.env` | Secret loading |
| Redis host | `127.0.0.1` | Chat and signal memory |
| Redis port | `6379` | Chat and signal memory |
| Redis auth | none | Redis connection |
| LLM base URL | `https://api.deepseek.com` | OpenAI-compatible client |
| LLM model | `deepseek-chat` | All chat completions |
| Bot log path | `/opt/hermes/logs/tg.log` | Rotating bot logs |
| Log rotation | `100 MB` | Loguru file rotation |
| Screenshots path | `/opt/hermes/workspace/screenshots` | `/screenshot` output |
| Default browser profile | `/opt/hermes/browser-data/default` | Shared browser state |
| Agent browser profile root | `/opt/hermes/browser-data/{agent_name}` | Intended per-agent browser state |
| Chromium executable | `/root/.cache/ms-playwright/chromium-1217/chrome-linux64/chrome` | Playwright browser launch |
| Task file | `/opt/hermes/tg-bot/autonomous_tasks.json` | Persistent autonomous task definitions |
| Telegram polling | enabled | Bot update transport |
| Autonomous interval | `300` seconds | Keyword and agent loop sleep |
| Signal memory TTL | `604800` seconds | Seven-day Redis dedupe retention |
