# Hermes 长期自主优化路线图

> 版本: v1.0 | 创建: 2026-05-24 | 负责人: Hermes Trading Agent

## 愿景

在模拟盘（Paper Trading）上构建稳定、可长期盈利的自动化加密货币交易系统。只有当模拟盘表现达到稳定正收益、可控回撤、合理胜率后，才考虑实盘。

---

## 迭代阶段（10 阶段）

| 阶段 | 名称 | 目标 | 状态 |
|------|------|------|------|
| **P1** | 回测框架 MVP | 历史 K 线回测 + 绩效指标 + CLI | ✅ 完成 |
| **P2** | 数据基础设施 | 多周期/多币种缓存、增量更新、数据质量检查 | ✅ 完成 |
| **P3** | 市场结构策略优化 | 走步验证 + Optuna 参数优化 + 基准对比 | ✅ 完成（多策略对比框架） |
| **P4** | 反馈引擎修复 | 修复 decision_feedback 评估、持久化 candidate stats | ✅ 完成（价格兜底+60s延迟） |
| **P5** | 情绪策略回测 | 合成/历史信号代理，复现 opportunity_engine 逻辑 | ✅ 完成（sentiment_proxy + sentiment_hybrid） |
| **P6** | 多策略并行 | 策略注册表、权重分配、冲突消解 | 🔄 进行中（自动迭代筛选） |
| **P7** | 模拟盘增强 | 滑点/手续费模型对齐、Telegram 绩效命令、告警 | ⏳ 待开始 |
| **P8** | 监控与告警 | 结构化日志、每日绩效报告、异常检测 | ⏳ 待开始 |
| **P9** | 7 天模拟盘验证 | 连续运行、每日分析、参数微调 | ⏳ 待开始 |
| **P10** | 实盘就绪评估 | 综合报告、风险清单、go/no-go 决策 | ⏳ 待开始 |

---

## 长期工作循环

```
分析表现 → 自动回测 → 优化参数/策略 → 模拟盘运行 → 记录 docs/performance_log.md → 重复
```

每个迭代周期（约 3-7 天）输出简短阶段总结。

---

## 成功标准（模拟盘 → 实盘门槛）

| 指标 | 目标 |
|------|------|
| 7 日滚动收益 | > 0% |
| 最大回撤 | < 15% |
| 胜率 | > 45% |
| 盈亏比 | > 1.2 |
| 夏普比率 | > 0.5 |
| 蒙特卡洛 ruin rate | < 5% |

## 架构演进

```
当前:  Telegram Bot + intel/ 情绪管道 + JSON 模拟盘
  ↓
P1-P3: backtest/ 量化模块 + 市场结构策略验证
  ↓
P4-P6: 多策略融合 + 反馈闭环
  ↓
P7-P10: 生产级监控 + 7天验证 + 实盘评估
```

---

## 风险与约束

1. **永远模拟盘** — 所有交易仅在 paper trading / testnet 模式
2. **密钥安全** — API Key 仅从 .env 读取，不入 Git
3. **文件规范** — 核心文件 ≤ 250 行，强模块化，中文注释
4. **自主迭代** — 除非重大架构变更或安全风险，无需人工确认
